# Подключение удалённого репозитория — пошаговая инструкция

Подробное руководство для начинающих. Каждый шаг объяснён.

---

## Что такое удалённый репозиторий?

**Репозиторий** — это папка с проектом, в которой Git хранит историю изменений (коммиты).

**Удалённый репозиторий (remote)** — копия вашего проекта на сервере (GitHub, GitLab и т.д.). Это нужно, чтобы:
- хранить код в облаке (бэкап);
- работать в команде;
- подключать облачного агента Cursor к актуальному коду.

**origin** — стандартное имя удалённого репозитория (можно назвать иначе, но так принято).

---

## Шаг 1: Создать аккаунт на GitHub (если ещё нет)

1. Откройте https://github.com
2. Нажмите **Sign up**
3. Зарегистрируйтесь (email, пароль, имя пользователя)

---

## Шаг 2: Создать новый репозиторий на GitHub

1. Войдите в GitHub
2. Нажмите зелёную кнопку **New** (или **+** → **New repository**)
3. Заполните:
   - **Repository name**: `tender-rf` (или другое имя)
   - **Description**: `AI-платформа для поиска и анализа тендеров РФ (44-ФЗ, 223-ФЗ)`
   - **Public** (публичный)
   - **НЕ** ставьте галочку "Add a README" — у вас уже есть код
4. Нажмите **Create repository**

После создания GitHub покажет страницу с URL репозитория, например:
`https://github.com/ВАШ_ЛОГИН/tender-rf.git`

---

## Шаг 3: Подключить удалённый репозиторий в терминале

Откройте терминал (PowerShell или CMD) в папке проекта и выполните команды **по очереди**.

### 3.1. Перейти в папку проекта

```powershell
cd c:\Projects\tender-rf
```

**Что это значит:** `cd` = "change directory" (сменить папку). Вы переходите в корень проекта.

### 3.2. Добавить удалённый репозиторий

```powershell
git remote add origin https://github.com/ВАШ_ЛОГИН/tender-rf.git
```

**Замените `ВАШ_ЛОГИН`** на ваш логин GitHub (например, `mila-dev`).

**Что это значит:**
- `git remote add` — добавить удалённый репозиторий;
- `origin` — имя (можно вызвать по имени);
- URL — адрес репозитория на GitHub.

### 3.3. Переименовать ветку в main (опционально)

Git по умолчанию создаёт ветку `master`. GitHub использует `main`. Чтобы синхронизировать:

```powershell
git branch -M main
```

**Что это значит:** `branch -M` — переименовать текущую ветку в `main`.

### 3.4. Отправить код на GitHub (первый раз)

```powershell
git push -u origin main
```

**Что это значит:**
- `git push` — отправить коммиты на удалённый сервер;
- `-u origin main` — связать локальную ветку `main` с удалённой `origin/main` (в следующий раз можно писать просто `git push`).

Git может запросить авторизацию:
- **Логин** — ваш логин GitHub;
- **Пароль** — используйте **Personal Access Token** (см. ниже), а не обычный пароль.

---

## Шаг 4: Personal Access Token (если Git просит пароль)

С 2021 года GitHub не принимает обычный пароль при `git push`. Нужен **токен**.

### 4.1. Создать токен

1. GitHub → **Settings** (ваш профиль) → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. **Generate new token** → **Generate new token (classic)**
3. **Note**: `tender-rf` (название для себя)
4. **Expiration**: 90 days или No expiration
5. Отметьте галочку **repo** (полный доступ к репозиториям)
6. **Generate token**
7. **Скопируйте токен** — он показывается один раз. Сохраните в надёжном месте.

### 4.2. Использовать токен

Когда Git спросит пароль — вставьте токен вместо пароля.

---

## Шаг 5: Проверить, что всё работает

1. Обновите страницу репозитория на GitHub — должны появиться файлы проекта.
2. В терминале выполните:

```powershell
git remote -v
```

Должно вывести что-то вроде:
```
origin  https://github.com/ВАШ_ЛОГИН/tender-rf.git (fetch)
origin  https://github.com/ВАШ_ЛОГИН/tender-rf.git (push)
```

## Полезные команды на будущее

| Команда | Что делает |
|---------|------------|
| `git status` | Показать изменённые файлы |
| `git add .` | Добавить все изменения в индекс |
| `git commit -m "описание"` | Создать коммит |
| `git push` | Отправить коммиты на GitHub |
| `git pull` | Скачать изменения с GitHub |

---

## Если что-то пошло не так

### Ошибка: "remote origin already exists"

Удалить старый remote и добавить заново:
```powershell
git remote remove origin
git remote add origin https://github.com/ВАШ_ЛОГИН/tender-rf.git
```

### Ошибка: "failed to push"

- Проверьте URL репозитория;
- Убедитесь, что репозиторий создан на GitHub;
- Используйте Personal Access Token вместо пароля.
